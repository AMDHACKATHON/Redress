## 1. Problem Statement

Most people have a legitimate complaint against a bank, telco, utility provider, landlord, or government agency but have no idea how to formally escalate it. They either give up, send an ineffective email, or don't know which regulatory body to contact. Redress solves that.

---

## 2. Product Summary

Redress is a web-based AI agent that helps anyone draft a formal complaint letter, find the right channel to send it, and escalate to the relevant regulatory body if ignored. It works globally, across any country and any sector, through a simple conversational interface.

---

## 3. Target Users

- Anyone who has been wronged by a bank, telco, utility, landlord, or government service
- Works for any country, any sector
- No technical knowledge required

---

## 4. Core User Flow

```
User registers / logs in
        │
        ▼
Creates a new complaint
        │
        ▼
  [Stage: understand]
  Describes problem in plain language
  Agent asks 2-3 clarifying questions
        │
        │ Agent has enough info
        ▼
  [Stage: draft]
  Agent generates formal complaint letter
  Letter displayed on screen + PDF download
        │
        │ User says complaint was ignored
        ▼
  [Stage: escalate]
  Agent generates escalation letter to regulator
  Regulator contact + filing instructions provided
  Escalation letter PDF download
```

## 5. Features

### 5.1 Authentication
- Email + password registration
- Login with session persistence
- Protected dashboard routes
- Logout

### 5.2 Complaint Management
- Create a new complaint session
- View all past complaints (history)
- View single complaint with full message thread
- Delete a complaint

### 5.3 Agent Chat
- Multi-turn conversational interface
- Agent detects complaint type and country from context
- Agent asks targeted clarifying questions
- Stage progression: understand → draft → escalate
- `ready_for_letter` signal triggers Generate Letter button
- `stage: escalate` signal triggers Escalate button

### 5.4 Letter Generation
- Formal complaint letter generated by AMD LLM
- Displays: full letter text, recipient, recommended channel, regulator details
- PDF download (client-side, jspdf)
- Letter saved to MongoDB

### 5.5 Escalation
- Escalation letter generated when initial complaint is ignored
- Addressed to relevant regulatory body
- Includes regulator contact and filing instructions
- PDF download
- Escalation letter saved to MongoDB

### 5.6 Profile
- View and update name, avatar, country
- View complaint count
- Delete account

### 5.7 Legal Pages
- Terms of Service page
- Privacy Policy page

## 9. Agent Behaviour

### Stage Logic
- Agent starts every complaint in `understand` stage
- Agent asks a maximum of 3 clarifying questions before moving to `draft`
- Agent sets `ready_for_letter: true` when enough context is gathered
- Agent moves to `escalate` when user indicates their complaint was ignored
- Stage is stored per complaint in MongoDB

### Letter Generation Prompt Requirements
- Formal, professional tone
- Includes: date, complainant details, recipient details, clear description, resolution requested, response deadline
- Tailored to complaint type and country

### Escalation Prompt Requirements
- References the original complaint and date sent
- Addressed to the regulatory body
- Requests intervention and resolution
- Includes filing instructions fetched via AMD web search tool or known regulator data

### Regulator Detection
- Agent identifies country from user context or asks
- Agent maps complaint type + country to the relevant regulatory body
- Contact details fetched where possible
